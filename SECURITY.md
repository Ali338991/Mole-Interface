# Security Policy

## Supported Versions

Mole is in early development. Security fixes are applied to the latest `main`.

## Reporting a Vulnerability

Please **do not** open a public issue for security vulnerabilities.

Instead, report them privately:

- Use GitHub's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) on this repository, or
- Email the maintainer at **[your-contact-email]**.

Please include steps to reproduce, affected versions, and any relevant logs.
You can expect an initial response within a few days. Thank you for helping keep
Mole and its users safe.

## Note on system access

Mole is designed to perform destructive file operations (cleanup, uninstall).
Reports about paths that could be deleted unsafely, bypasses of the Trash-routing
or dry-run safeguards, or privilege-escalation concerns are especially valued.
